Java学习笔记(八) - 继承

MeetMax 11 Sep 2018 on Java

一、概念

一个类派生字另一个类被称为子类，派生的类称为超类，也称为基础类或父类。在Java中，Object是最高等级的类，所有类都是它的子类，除了Object类没有父类，每个类都有一个且仅有一个直接的父类，Java仅支持单继承，在没有任何显式的超类的情况下，每个类都是Object的隐式的子类。子类继承父类的索引成员(字段，方法和嵌套类等)，构造方法不是成员，因此没有被子类继承，但是父类的构造方法能够在子类中调用，通过super关键词。
下面声明一个Bicycle的类

public class Bicycle {
        
    // the Bicycle class has three fields
    public int cadence;
    public int gear;
    public int speed;
        
    // the Bicycle class has one constructor
    public Bicycle(int startCadence, int startSpeed, int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;
    }
        
    // the Bicycle class has four methods
    public void setCadence(int newValue) {
        cadence = newValue;
    }   
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
        
}

声明MountainBike类继承Bicycle类：

public class MountainBike extends Bicycle {
        
    // the MountainBike subclass adds one field
    public int seatHeight;

    // the MountainBike subclass has one constructor
    public MountainBike(int startHeight,
                        int startCadence,
                        int startSpeed,
                        int startGear) {
        super(startCadence, startSpeed, startGear);
        seatHeight = startHeight;
    }   
        
    // the MountainBike subclass adds one method
    public void setHeight(int newValue) {
        seatHeight = newValue;
    }   
}

MountainBike类将继承Bicycle的所有字段和方法，构造函数不继承，在子类中可以通过super关键词调用。

二、子类可以做什么？

一个子类将继承它的父类的所有public和protected方法，无论这个子类在哪个包，如果子类和父类在同一个包，它也将继承所有package-private的成员，此外，子类还满足以下规则：

  • 继承的字段可以直接被使用
  • 在子类中可以声明一个和父类相同名称的字段，此时父类的字段不会被覆盖，而是被隐藏，如果在子类中要访问父类的同名字段，需要借助super关键词。
  • 你可以声明新的字段在子类中，改字段不会存在于父类中。
  • 继承的方法能被直接使用
  • 你可以在子类中声明一个实例方法，它和父类中的其中一个实例方法具有相同的方法签名，注意是实例方法，不是静态方法，此时，父类的该方法将被重写，且也能通过super关键词被访问到。
  • 你可以在子类声明一个静态方法，它和父类中的一个方法具有相同的方法前面，此时，父类的该方法不会被覆盖掉，只是被隐藏了，在子类中想要访问父类的该方法，可通过super关键词调用。
  • 你可以在子类中声明新的方法，它不在父类中。
  • 你可以再子类中写一个构造函数，它可以隐式的或者显示的调用父类中的构造函数，使用super关键词。

三、对象转换

每个对象都有对应的类型，假如声明一个MountainBike的对象mBike，那么该对象就是MountainBike类型，如下：

MountainBike mBike = new MountainBike();

由于MountainBike继承自Bicycle类，因此它也可以是Bicycle类型，如下：

Bicycle mBike = new MountainBike();

一个对象可以被转换成它集成的类或者实现的接口类型，例如任何类都可以转换成Object类型，如下：

Object obj = new MountainBike();

如果我们写成 MountainBike mBike = obj将会编译错误，因为编译器不知道obj是MountainBike类型的，然而，我们可以告诉编译器，将obj显式转换成MountainBike类型，如下：

MountainBike myBike = (MountainBike)obj;

这时候编译器会做运行时检查，如果编译器确定obj被赋予了MountainBike类型的值，则编译通过。

四、重写和隐藏方法

1. 实例方法

如果在子类中声明一个在父类中已存在的实例方法，即它们的方法签名相同，此时，子类的方法将重写父类的方法。在重写方法时 ，你可以使用@Overriding注解，它的好处是，如果父类不存在该方法，编译器会报错，可以避免一些人为的错误。

2.静态方法

如果在子类中声明一个父类中已存在的静态方法，那么父类中的方法将被隐藏。在子类中需要通过super关键词访问父类的静态方法。

3.接口方法

在接口中的默认方法和抽象方法和实例方法一样都能被继承，然而，当父类或者接口中有多个拥有相同方法签名的方法时，Java编译器遵循继承规则解决名称冲突，这些规则遵循以下两条原则：

  • 实例方法的优先级高于接口的默认方法
  • 已经被其它候选项重写的方法将被忽略，这种情况通常发生在，子类实现的多个接口拥有共同的父级接口时发生，请看下面的例子：
public interface Animal {
    default public String identifyMyself() {
        return "I am an animal.";
    }
}
public interface EggLayer extends Animal {
    default public String identifyMyself() {
        return "I am able to lay eggs.";
    }
}
public interface FireBreather extends Animal { }
public class Dragon implements EggLayer, FireBreather {
    public static void main (String... args) {
        Dragon myApp = new Dragon();
        System.out.println(myApp.identifyMyself());
    }
}

The method Dragon.identifyMyself returns the string I am able to lay eggs.
